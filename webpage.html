WCAG Accessibility

WCAG is divided into 4 categories with a total of 12 practices:

Perceivable

  • 1.1 Provide text alternatives
  • 1.2 Provide alternative methods
  • 1.3 Provide the ability to present content in different ways, allowing for customization
  • 1.4 Make it easier to see and hear content

Operable

  • 2.1 Make functionality available via the keyboard
  • 2.2 Provide enough time to interact
  • 2.3 Avoid designing/developing content that causes seizures
  • 2.4 Help navigate, find content, and determine where they are focused

Understandable

  • 3.1 Text content should be understandable
  • 3.2 Interface should be consistent and predictable
  • 3.3 Help users fix mistakes

Robust

  • 4.1 Maximize compatibility of user agents and assistive technologies

Media

Media accessibilty can be achieved using captions and transcripts for the visually or hearing disabilities.

Nowadays, web designers may decide to utilize autoplay, but this doesn't comply with WCAG 1.4.2.

  • Audio must not automatically play for more than 3 seconds.
  • Must be able to pause or stop it.
  • Must be able to reduce the volume independently of system audio.
Some solutions to autoplaying audio are:
  • Make the audio shorter than 3 seconds.
  • Add a link at the top of the page or sticky positioned stop/pause or control volume.
  • Allow the user to initiate audio or video themselves.

Video

For videos, captions are used to describe anything audible for the hard of hearing. This can be done with a caption track by using a track tag. 

                
                < video controls src='video.mp4' >
                  < track kind='captions' src='captions.vtt' >
                < / video >
To add caption tracks, WebVTT formatted transcripts are used. In addition to captions, transcripts are also additions necessary for media accessibility.

Line Graphs

This image is a bad example of an accessible line graph in that it fails the following:

  • Being friendly towards the visually impaired, such as people that are colorblind, who may not be able to distinguish between the two different lines on the graph.
  • Does not provide a legend to reduce the cognitive burden.
  • Does not clearly label the x and y axis of the graph
There are resources out there that can provide better accessibility for line graphs, such as High Charts. Certain resources can allow for line trends to be focusable as opposed to inserting a large description to describe the graph, which can be initimidating and difficult to absorb.
